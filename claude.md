# SHOR. プロジェクト指示

あなたはこのプロジェクトのコードとドキュメントの整合性を保つ担当です。

## 最重要ルール：コードとドキュメントを常に一致させる

- コード（index.html など）を変更したら、対応する docs/ 内のドキュメントも
  同じ内容になるよう必ず更新すること。
- 逆に、docs/ 内の仕様を変更するよう指示されたら、コードもそれに合わせて更新すること。
- ドキュメントを更新したときは、「どのファイルの、どこを、なぜ変えたか」を
  作業の最後に一言で報告すること。

## コードと docs の対応表

- 写真の配信・マッチングのロジック      → docs/distribution.md
- 閲覧権（無料枠・投稿枠、1日2枚まで）   → docs/view-grants.md
- 現像インタラクション（長押し・ぼかし）  → docs/develop-interaction.md
- データ構造（User / Post / ViewHistory）→ docs/data-model.md
- 画面の種類と遷移                      → docs/screens.md

## 迷ったときの原則

- どの docs を直すべきか判断がつかないときは、勝手に決めず、私に確認すること。
- ドキュメントを大きく書き換える場合は、変更後の内容を先に見せて、私の承認を得てから確定すること。
